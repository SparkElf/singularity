## 1. 目标与范围

P1-B4使浏览器编辑器不再依赖思源旧应用壳的布局、页签、Dock、搜索、历史、卡片、插件注册表和移动端实现，同时保持用户现有编辑、菜单、浮窗、状态统计、全屏、面板刷新和错误反馈行为。完成后，React工作台可以通过公开运行时合同装配Protyle，不需要构造或暴露旧`App`对象。

本批不改变编辑器视觉、块事务、插件公开行为、快捷键、内容格式或Kernel API；不实现企业权限、真实Gateway、评论或实时协作。P1-B4包含可替换的P2传输端口、会话状态机和错误类别，但真实HTTP授权、空间路由及Gateway序列化证据留到P5；在P5通过前不得声称真实空间隔离或L0完成。浏览器不得直连未来的空间Kernel地址，也不得保留旧请求路径作为fallback。

## 2. 同类调研

### 2.1 思源3.7.1本地官方源码

锁定基线`41f2861c87575ff5ac4b50a0520b1a4fe55b4a70`是本批兼容事实源。当前Protyle通过完整`App`、旧布局扫描、全局菜单与浮层集合、`fetchPost`错误处理链和工作台工具函数完成编辑器外部动作。优点是功能完整；缺点是任一通用请求或菜单调用都会把旧布局、移动端和Electron模块带入浏览器闭包，React无法独立装配编辑器。

本批借鉴其已经验证的用户行为和操作顺序，不继承其全局所有权。菜单与块DOM操作仍贴近编辑器，页签、Dock、面板和全屏工作台状态由宿主拥有，请求与订阅由单空间会话拥有。

### 2.2 奇点现有边界

P1-B1已经用类型化事件承接文档、搜索、图谱、历史、卡片、资源和关闭动作；P1-B2以插件端口保留插件顺序和扩展字段；P1-B3以会话注册表统一活动编辑器和跨编辑器协同。B4在这些既有边界上补齐运行时装配、传输、菜单浮层和工作台状态，不建立第二套同义合同。

### 2.3 Tiptap编辑器生命周期

Tiptap把可编辑性作为显式状态，提供`setEditable()`切换，并用`destroy()`解绑全部事件、`isDestroyed`判断终止状态。优点是实例生命周期清楚，框架可安全重建；缺点是它不定义企业空间、传输错误或跨编辑器浮层所有权，不能直接替代奇点Session。

本批借鉴“显式可编辑状态”和“销毁即终态”：`readOnly`由宿主输入，编辑器销毁后任何迟到请求、订阅或回调都不能恢复实例。不会引入Tiptap内容模型或复制其API命名。

### 2.4 CKEditor只读与崩溃恢复

CKEditor以独立锁控制只读，未改变内容的搜索和导出仍可使用；Watchdog拥有编辑器创建/销毁，在崩溃时重建实例，并区分`ready`、`crashed`、`crashedPermanently`和`destroyed`。优点是权限只读与运行故障分离、生命周期状态可观测；缺点是自动重建会复制编辑器内容快照，不符合奇点由Kernel持有内容事实的约束。

本批借鉴两点：权限`readOnly`不能被网络恢复或插件意外解除；编辑器所有者必须掌握创建、销毁和迟到结果。奇点不复制正文到Watchdog，不自动重建永久失败实例，也不把崩溃恢复作为权限fallback。

## 3. 用户故事

- 作为知识库用户，我在React工作台中继续使用编辑、菜单、浮窗、全屏、字数统计、大纲和反链，不因旧壳移除而失效。
- 作为知识库用户，我遇到网络、权限或Kernel错误时能看到明确且一致的错误结果，不出现静默失败、整页刷新或旧客户端提示。
- 作为知识库用户，我关闭浮窗、切换文档或切换空间后，旧菜单、请求、订阅和面板回调不能继续影响当前界面。
- 作为插件作者，我的编辑器插件仍只通过既有插件合同工作，不因运行时收口改用另一套字段或调用顺序。
- 作为平台维护者，我可以从公开浏览器入口追踪完整运行时闭包，确认其中没有旧`App`、Electron、Node内置或原生移动实现。

## 4. 交互与状态

### 4.1 编辑器装配

创建编辑器时，当前空间会话提供`spaceId`、宿主动作、插件、编辑器注册、请求、订阅、菜单和浮层能力；创建者另提供`documentId`、`readOnly`和表面类型。`readOnly`是Factory/Controller边界，不进入可被插件合并的Protyle options；权限或错误把它收紧为`true`后，只有宿主取得新的授权事实才能恢复为`false`。编辑器只能使用这些明确能力，不能穿透访问工作台对象。正文、搜索、反链和浮层使用同一空间运行时；只读历史预览使用同一配置和本地渲染能力，但不加入活动会话或实时订阅。

### 4.2 工作台协同

编辑器请求大纲定位或刷新、反链刷新、文档元数据更新、全屏、布局保存、活动状态和字数统计时，由当前宿主处理。事件只携带目标文档、动作和最小结果，不携带布局模型、DOM、完整编辑器或正文。

正文内部菜单、选区、块事务和属性编辑保持就地交互；同一空间的菜单与浮层只有一个会话所有者。权限`readOnly`关闭所有写事务和影响内容的插件命令，但保留读取、搜索、复制及不修改内容的操作；网络恢复不能解除权限只读。

### 4.3 请求与订阅

所有Kernel HTTP与WebSocket操作经当前空间会话执行。B4以可控的协议边界证明请求取消、订阅终止和错误分类；P5再以真实HTTP/WS Gateway证明认证、授权与空间路由。会话结束后，迟到响应和重连不得更新界面。

错误结果固定如下：

| 类别 | 用户可见结果 | 当前内容 | 后续动作 |
| --- | --- | --- | --- |
| 未认证 | 编辑区保持但禁止继续写入，工作台显示会话失效 | 保留已渲染内容 | 重新登录；不得自动刷新 |
| 无权限 | 编辑器切为只读，工作台显示无权修改 | 保留已渲染内容 | 返回或请求授权；不得重试写入 |
| Kernel不可用 | 编辑器停止提交并显示内容服务不可用 | 保留已渲染内容 | 用户显式重试；不得切换地址 |
| 网络失败 | 当前操作显示未完成，编辑器停止自动重复写入 | 保留已渲染内容 | 用户显式重试；不得默认成功 |

### 4.4 生命周期状态

- **关闭嵌入式所有者**：先中止该所有者请求，再销毁其子编辑器和浮层；迟到响应不得重新创建或登记实例。
- **切换文档**：先终止旧文档编辑器的请求与订阅并销毁实例，再创建新文档实例；旧文档回调不得更新新文档。
- **切换空间**：先停止旧Session接收新工作，依次中止请求、关闭订阅、销毁浮层/菜单和编辑器，最后释放插件端口；新Session不复用旧集合或回调。
- **主动销毁**：销毁是终态；网络恢复、插件回调和迟到Promise均不能使实例重新进入Registry。

### 4.5 边界状态

编辑器表面只区分工作台正文和嵌入式表面；该状态由创建者明确提供，不根据DOM类名、是否有布局model或是否加入Registry推断。工作台表面可请求页签、全屏和面板动作；嵌入式表面只处理自身DOM与显式宿主动作。

## 6. 约束

- **内容所有权**：正文、选区、事务和撤销栈仍只属于Protyle与Kernel，不进入React或Zustand。
- **空间隔离**：请求、订阅、菜单、浮层和Registry均属于单一`spaceId`会话；切空间先完整销毁旧会话。
- **权限**：浏览器只表达只读体验，HTTP与WebSocket写权限最终由Gateway强制执行。
- **错误**：不新增刷新、降级、旧请求、直连Kernel或默认成功fallback。
- **字段**：编辑器装配只使用`spaceId`、`documentId`、`readOnly`、表面和动作所需最小字段；不增加`workspaceId`、`rootId`等同义字段。
- **性能**：工作台事件不携带正文或DOM；Registry、菜单和浮层按实例增量管理，不扫描完整布局树。
- **兼容**：以本仓思源3.7.1浏览器行为为准；不承诺旧Electron独立窗口和原生移动端入口。
- **i18n与视觉**：不改变页面布局和视觉规则；错误类别先以宿主状态合同交付，P3/P5显示文案时必须进入既有语言资源并继续使用设计系统。
- **测试**：B4永久门禁进入`cd enterprise && pnpm lint && pnpm typecheck && pnpm test && pnpm build`；旧App只运行项目既有定向入口，后续浏览器测试进入`enterprise/apps/web`标准Playwright入口。不以源码字符串、完整内部mock或静态壳冒充真实编辑器行为。
- **产物**：日志、文档、fixture和测试输出只记录标识、阶段、结果与必要原因，不记录正文、选区、令牌或内部提示。
