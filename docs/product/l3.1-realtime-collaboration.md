# 奇点 L3.1 生产实时协作产品需求

> L3.0 语义原型和 L3.1 生产链路均已通过集中验证。本阶段把已验证的操作语义接入生产 React/Protyle、NestJS Gateway 和 Go Kernel；功能开关仍默认关闭，按内部试用和灰度合同开放。

## 1. 目标与边界

### 1.1 本期包含

- 桌面 Web 上两个或更多编辑者同时打开同一文档并编辑。
- 普通文本、标题、列表/容器块、块引用、嵌入块和属性视图单元格的增量操作。
- 服务端确认、乱序/重复消息幂等、短暂断线恢复、自己的操作撤销和可观察冲突。
- 在线状态、光标和选区作为临时投影，不进入正文历史。
- 文档级 ACL 复用 L2 `inherit/restricted` 和 `viewer/commenter/editor` 能力；撤权立即关闭实时会话。
- 单个 API 副本和单个空间 Kernel 部署下的生产 WebSocket 协作。
- 受限加密内容库协作：仅已登录组织成员、显式文档授权和当前内容密钥可用时才能加入；匿名分享、公开协作和未授权密钥转发均不允许。
- 功能开关、内部试用、灰度指标、关闭开关后的只读收敛和回滚。

### 1.2 本期不包含

- 多副本 Gateway 的跨实例消息总线、跨区域协作和自动水平扩展。
- 移动端生产体验、离线无限编辑、跨文档原子操作和附件二进制并发。
- 插件私有状态、复杂属性视图 schema 迁移和整篇文档快照同步。
- 匿名或公开分享直接加入实时协作。
- 服务器无法解密的零知识端到端实时协作协议。若部署密钥不允许 Go Kernel 在授权会话内处理内容，本阶段返回明确的 `encrypted-collaboration-unavailable`，不降级为明文或快照路径。
- 锁住整篇文档、最后写入获胜、静默丢弃已确认操作或整篇正文覆盖。

## 2. 本地先例与决策依据

本轮不进行非必要网页搜索。仓库已有 L1/L2 合同和 L3.0 验证证据足以冻结本期产品边界：

| 本地来源 | 已确认事实 | L3.1 借鉴 | 不直接照搬 |
| --- | --- | --- | --- |
| [L3.0 PRD](l3-realtime-collaboration.md) 与验证报告 | 语义操作、四段身份、冲突、撤销和恢复已在隔离原型通过 | 将用户可观察语义带入生产合同 | 不把原型协调器当生产服务 |
| [L3.0 架构](../architecture/l3-realtime-collaboration.md) | Go Kernel 是内容事实源，TypeScript 只承载协议和浏览器壳 | 保持单一 reducer 和最小增量 | 不复制正文到 PostgreSQL |
| [L2 PRD](l2-async-collaboration.md) 与 [ADR-029](../adr/0029-l2-async-collaboration-boundary.md) | ACL、审计、组织空间身份属于 Nest 控制面 | 复用现有 ACL owner 和撤权事件 | 不在实时模块重做权限算法 |
| [ADR-006](../adr/0006-realtime-collaboration-gate.md) | WebSocket 广播不等于并发语义 | 使用独立双向协作协议 | 不改造只接收推送的 Protyle WS |
| [ADR-018](../adr/0018-cross-process-access-change.md) | 单副本连接注册表和 PostgreSQL 提交通知已有先例 | 首期复用单副本生命周期 | 不提前引入 Redis/NATS |

## 3. 角色与用户故事

### 3.1 角色

- **编辑者**：拥有当前文档 `editor` 能力，可以提交语义操作并撤销自己的已确认操作。
- **观察者**：拥有 `viewer` 或 `commenter` 能力，只接收允许展示的正文结果和临时状态。
- **空间管理员**：可修改文档 ACL、关闭协作和处理受限加密库成员授权。
- **加密库成员**：通过显式成员关系获得内容密钥使用权；密钥不可出现在浏览器日志、控制面审计或协作消息中。
- **运维/发布者**：控制功能开关、灰度范围、容量上限和回滚。

### 3.2 用户故事

- 作为两名编辑者，我可以同时修改同一文档，双方最终看到相同结果，已确认操作不会静默丢失。
- 作为编辑者，我可以撤销自己的操作，但不会撤销其他人的后续编辑。
- 作为观察者，我能看到授权后的结果，但不能通过伪造浏览器消息提交正文操作。
- 作为空间管理员，我撤销文档权限后，旧会话立即停止写入和接收推送；恢复权限后必须重新加入。
- 作为加密库成员，我只有在仍拥有成员授权和可用密钥时才能协作；失去任一条件时看到明确的受限结果。
- 作为发布者，我可以关闭功能开关，已连接会话进入只读/关闭状态，不切换到备用地址或隐式降级。

## 4. 交互与状态

### 4.1 加入协作

1. 当前 React Session 提供 `organizationId + spaceId + notebookId + documentId` 和当前能力。
2. 客户端使用相同四段身份发起专用协作会话；不能从 DOM、首个响应或全局内容库推断身份。
3. 服务端重新确认 ACL、功能开关、内容安全模式和 Kernel 可用性后返回 canonical 版本。
4. 会话状态只有 `connecting`、`ready`、`reconnecting`、`conflict`、`revoked`、`closed`；状态变化可被用户观察。

### 4.2 并发编辑和冲突

- 正文变更以最小语义操作提交，不发送整篇最终正文。
- 服务端确认后广播 canonical 操作；重复操作返回幂等结果，迟到或跨身份消息被拒绝且不改变正文。
- 可合并操作自动收敛；同一单元格并发写入、同一块并发移动等不可无损合并的结果进入 `conflict`，显示冲突对象和可执行处理结果。
- 冲突未解决时不使用 LWW、锁或整文档覆盖掩盖差异。

### 4.3 断线、撤权和关闭

- 断线进入 `reconnecting`，未确认正文操作不继续提交；恢复使用最近确认的因果位置。
- ACL 撤销、加密密钥失效、功能开关关闭或 Kernel 不可用分别进入明确状态；不伪造成功，不切换其他文档或空间。
- 单副本进程重启只保留 Kernel 的 canonical 历史；presence 清除，客户端重新加入并从历史缺口恢复。

### 4.4 加密内容库

- 加密库默认不开放实时协作，管理员必须显式启用“受限协作”。
- 加入者必须同时满足：组织成员有效、文档 ACL 允许、内容密钥可用、客户端版本满足协作协议。
- 协作服务器不把密钥写入 PostgreSQL、审计、日志或浏览器持久化存储；Go Kernel 在授权内容会话内完成解密和语义应用。
- 若密钥不可用，用户看到“该加密内容暂不可协作”，而不是明文、旧快照或备用通道。

## 5. 验收标准

每条标准以用户可观察结果或稳定业务合同判定；源码字符串、私有函数、完整内部 mock 或固定截图不能单独作为通过证据。

| 编号 | 验收标准 | 失败风险 | 最低充分证据 |
| --- | --- | --- | --- |
| L31-SEM-01 | 两个编辑者并发修改普通块后最终结果一致，所有已确认操作都可追踪 | LWW/覆盖写丢数据 | 真实 Kernel + 协议 integration |
| L31-SEM-02 | 文本、块父子关系、引用、嵌入和 AV 单元格在并发后仍可解析 | 内容语义损坏 | Kernel semantic integration |
| L31-HIS-01 | 刷新、断线重连和单副本重启后结果可从 canonical 历史恢复 | 只存在内存状态 | Kernel history + E2E |
| L31-HIS-02 | 撤销只生成当前客户端自己的逆操作，不撤销他人后续操作 | 误撤销他人内容 | multi-client integration |
| L31-CON-01 | 不可无损合并的移动、单元格或引用冲突进入明确冲突状态，不静默选值 | 冲突丢失 | protocol + browser integration |
| L31-ID-01 | join、submit、resume、presence、确认和撤权事件均携带四段内容身份 | 跨空间/跨库污染 | contract + protocol integration |
| L31-ACL-01 | viewer/commenter 无法提交正文；撤权后旧会话不能写入或接收推送 | 客户端自证权限 | 真实 HTTP/WS integration |
| L31-ENC-01 | 未启用受限协作的加密库不能加入；密钥不可用时返回明确受限结果 | 加密内容泄露或假协作 | security + Kernel integration |
| L31-ENC-02 | 加密库协作不会把密钥或正文写入控制面、日志或浏览器持久化存储 | 敏感信息泄露 | logger/storage audit |
| L31-REC-01 | 重复、乱序、迟到和断线消息不会重复写入或覆盖其他文档 | 竞态和重复写 | fault-injected integration |
| L31-OPS-01 | 单副本启动、关闭、重启和功能开关变化都能得到可观察状态，presence 不残留 | 假在线/资源泄露 | lifecycle integration |
| L31-OPS-02 | 达到操作大小、并发人数和消息频率上限时返回明确限流/关闭结果，不无限排队 | 资源耗尽 | performance + protocol |
| L31-OBS-01 | join、operation、resume、conflict、revocation 和 lifecycle 日志带稳定关联与完整异常堆栈，且不含正文/密钥 | 无法诊断或泄密 | logger integration |
| L31-ROLL-01 | 关闭功能开关后新会话被拒绝，旧会话收敛到只读/关闭，重新开启后需重新授权 | 灰度回滚失控 | browser + API integration |

## 6. 约束

- **事实源**：Go Kernel 拥有正文、块、AV 和历史；Nest/PostgreSQL 只拥有 ACL、开关、审计和会话元数据。
- **身份**：所有内容消息显式携带组织、空间、笔记本和文档身份；不从近似字段推断。
- **权限**：L2 `DocumentAccessPolicy` 是唯一 ACL owner；协作模块只消费已授权 capability 和撤权事件。
- **部署**：首期单 API 副本、单空间 Kernel 协作通道；不引入跨副本消息总线。
- **安全**：专用双向协作 WS 与只接收推送的 Protyle WS 分离；日志不记录正文、令牌或密钥。
- **前端**：沿用 React 19、Vite 8、Tailwind 4、shadcn、Zustand 和 React Query；跨组件会话状态使用 Zustand，正文仍由 Protyle/Kernel 事实流拥有。
- **设计系统**：协作状态、冲突、受限加密和只读状态先定义语义 token 与 shadcn variants，再由页面消费。
- **国际化**：用户可见状态和错误走现有语言资源，不暴露内部模块名。

## 7. 阶段门禁

L3.1 已完成 PRD、架构方案、ADR 和实施计划，并通过集中 code-review、test-governance 复评及唯一 `pnpm verify:l3-production` aggregate。功能开关仍默认关闭，开放前必须满足内部试用、容量观测和回滚条件；验证证据见 [L3.1 验证报告](../verification/l3.1-realtime-collaboration.md)。
