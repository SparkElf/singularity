## 1. 目标与范围

P1-B3统一当前空间内正文、搜索、反链和浮层编辑器的会话协同，使焦点、撤销、拖拽、内容更新、尺寸调整和关闭行为始终作用于正确且仍存活的编辑器。切换空间或结束会话后，旧空间编辑器不得继续响应任何操作。

本批覆盖浏览器工作台中可交互编辑器的进入和退出会话、焦点切换、跨编辑器拖拽、内容更新、尺寸调整、全局撤销及文档关闭。`CB_GET_HISTORY`只读历史与资源预览不接收协同事务、不作为活动编辑器且由旧所有者直接替换DOM，因此不加入会话。属性面板在脱离DOM容器中构造的ghost编辑器同样不加入会话。大纲与反链面板刷新、全屏工作台布局、全局浮层收起、React标签页管理、旧Electron独立窗口和原生移动端入口不在本批范围。

## 2. 同类调研

### 2.1 思源3.7.1本地官方源码

当前官方基线通过`getAllEditor`遍历布局、搜索、反链、自定义编辑器、对话框和`blockPanels`，通过`getAllModels().editor`处理正文页签，并通过DOM活动Tab选择撤销目标。优点是覆盖旧壳全部容器；缺点是编辑器身份依赖布局实现，同一能力在多个全局数组中重复扫描，销毁和切空间没有统一边界。

现有真实调用证明同一会话必须覆盖正文、搜索、反链、自定义编辑器和浮层，并在编辑器关闭及空间切换后立即停止旧实例参与协同。

### 2.2 现有奇点Session合同

ADR-009与ADR-010已经确定一空间一会话，切空间先结束旧会话。P0已定义编辑器重建生命周期；B3只补齐会话内协同，不改变React工作台的页面和标签状态。

### 2.3 外部核对限制

本轮沿用P1-B2已记录的网络限制：官方GitHub Raw只读访问发生TLS失败。由于本仓包含同版本完整官方布局、编辑器、事务和撤销实现，产品结果以本地官方基线为准，不引用未取得的外部资料。

## 3. 用户故事

- 作为用户，我在两个页签打开同一文档时，拖拽和Kernel推送能在正确实例中找到源块，不产生错误的撤销锚点。
- 作为用户，我在搜索、反链或浮层编辑器中聚焦后执行撤销，系统作用于当前聚焦实例，而不是旧布局中的另一个Tab。
- 作为用户，我关闭页签或浮层后，已销毁编辑器不再参与后续事务、resize或Host关闭扫描。
- 作为用户，我调整分屏或窗口尺寸后，各可见编辑器保持原先的视口锚点。
- 作为平台维护者，我切换空间或销毁Session后，旧空间实例不能进入新空间注册表。

## 4. 交互与状态

### 4.1 进入与离开会话

可交互编辑器就绪后加入当前空间会话，关闭后立即离开。重复的就绪或关闭通知不得造成重复响应或错误；当前编辑器关闭后，系统不擅自把撤销等操作转移到另一个未聚焦编辑器。只读历史预览只展示历史内容，不获得拖拽、事务扫描、尺寸协同或全局撤销资格。

### 4.2 当前编辑器

用户聚焦正文、搜索、反链或浮层编辑器时，该编辑器成为当前操作目标。已关闭编辑器的迟到事件不能恢复其当前状态；首次打开编辑器且尚未发生焦点切换时，全局命令仍有明确目标。

### 4.3 跨编辑器协同

拖拽必须依据用户实际拖起内容所在的编辑器确定源文档；同一文档打开多份时，服务端内容更新应用到仍存活且包含目标内容的实例；调整分屏或窗口尺寸时，只处理当前可见编辑器。无法确认拖拽来源时中止操作并给出可诊断错误，不猜测来源。

### 4.4 Session销毁

结束会话时，所有旧编辑器立即失去协同资格。旧编辑器的迟到加载、焦点或网络回调不得进入新空间；新空间建立独立会话。

## 5. 验收标准

| --- | --- | --- | --- |
| P-B3-01 | 同一编辑器无论收到多少次就绪或关闭通知，都只参与一次会话协同，关闭可重复执行且不报错 | 重复事务和资源泄漏；打开与关闭 | 自动生命周期验证 |
| P-B3-02 | 关闭的正文、搜索、反链或浮层编辑器不再响应内容更新、拖拽、尺寸调整或关闭请求 | 旧实例误响应；所有跨编辑器操作 | 自动生命周期验证 + 浏览器集成 |
| P-B3-03 | 首次打开即有明确操作目标；用户切换焦点后撤销作用于新目标；关闭当前编辑器后不擅自改投其他编辑器 | 撤销命中错误文档；焦点与关闭 | 自动状态验证 + 浏览器集成 |
| P-B3-04 | 切换空间或结束会话后，旧编辑器的迟到加载、焦点和网络回调均不能影响新空间 | 跨空间内容污染；空间切换 | 自动生命周期验证 + 浏览器集成 |

| P-B3-06 | 同一文档打开多份时，内容更新能命中仍存活且包含目标内容的正文或浮层实例 | 多实例内容不同步；服务端推送 | 边界检查 + 浏览器集成 |
| P-B3-07 | 调整分屏或窗口尺寸后，各可见编辑器保持视口锚点，已关闭或隐藏实例不参与调整 | 滚动位置跳变；布局调整 | 自动状态验证 + 浏览器集成 |
| P-B3-08 | 在正文、搜索、反链和浮层之间切换焦点后，全局撤销始终作用于用户当前编辑器 | 撤销目标错误；焦点切换 | 自动状态验证 + 浏览器集成 |
| P-B3-09 | 服务端要求关闭文档时，只关闭当前会话中仍打开的对应实例，不重复处理已关闭实例 | 关闭错误页签或重复销毁；服务端关闭事件 | 边界检查 + 浏览器集成 |
| P-B3-10 | 大纲、反链面板刷新、全屏布局、全局浮层和React标签页行为维持现状，不因本批协同改造发生回退 | 批次越界导致工作台行为回退 | 架构审查 + 后续浏览器回归 |
| P-B3-11 | 只读历史与资源预览不成为当前编辑器，也不参与拖拽、内容更新、尺寸调整和关闭扫描 | 预览替换DOM后留下失效会话引用；历史与资源预览 | 边界检查 + 后续浏览器回归 |

## 6. 约束

- **空间隔离**：编辑器只能参与其所属空间的当前会话，旧空间事件不能影响新空间。
- **内容所有权**：会话协同不得复制或另存文档正文、事务、撤销数据或页面状态。
- **性能**：拖拽、内容更新和尺寸调整不得因会话协同产生与文档正文大小相关的额外处理。
- **错误**：来源不明或会话已结束时显式中止并保留稳定诊断，不猜测来源或静默改投其他编辑器。
- **安全**：本批不改变Gateway权限，编辑器协同信息不得向插件或页面暴露可篡改的内部会话状态。
- **兼容**：React标签页、大纲、反链面板、全屏和全局浮层行为保持现有产品语义。
- **入口**：本批只覆盖`BROWSER=true`、`MOBILE=false`的浏览器工作台过渡入口；不声明旧Electron独立窗口或原生移动端兼容。
- **测试价值**：自动验证保护会话状态和边界合同；真实拖拽、内容更新、尺寸调整和焦点路径由P3浏览器集成验证。
- **产物**：文档、日志和测试产物只记录实例ID、文档ID、阶段和结果，不记录正文、选区或内部提示。
