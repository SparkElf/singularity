## 1. 目标与范围

S1交付可部署的企业身份启动切片：基础设施管理员通过部署主机上的受控运维入口配置本地账号、组织、空间与成员授权，本地账号成员登录后只能看到自己有权访问的组织与空间，并从服务端确认的空间状态进入工作台。路由中的组织和空间标识只用于定位请求，不能单独证明用户有权进入该空间。

本批包含受控首次初始化、最小账号与空间授权配置、本地账号登录、当前会话退出、会话到期与撤销、用户停用、组织与空间成员授权、授权空间列表、空间选择与切换、空间启动状态、不可见资源保护和权威角色展示。受控运维入口是S1的正式生产路径，但不属于普通成员Web界面。

本批不包含浏览器初始化、开放注册、邀请、找回或修改密码、MFA、OIDC、设备会话管理页面、用户组、完整成员管理页面、分享、审计、备份、Kernel Gateway、Protyle内容编辑和实时协作。上述能力不得以不可操作按钮、虚假数据或面向开发者的占位文案出现在S1界面。

### 1.1 已锁定产品决策

1. 新部署只允许基础设施管理员从部署主机的受控运维入口创建首位本地账号；成功时同时建立首个组织、首个空间及对应所有者权限，浏览器与公网API均不提供初始化入口。
2. 登录后提供授权空间入口。只有一个可访问空间时直接进入；存在多个空间时显示可搜索的空间列表；没有可访问空间时显示明确空状态。
3. 用户主动退出只撤销当前会话。基础设施管理员从受控运维入口停用用户或明确撤销该用户会话时，撤销该用户全部会话；普通成员不能查看或管理其他设备会话。
4. 未登录深链在登录成功后返回原同源目标；目标已失效或不可见时进入统一的空间不存在状态，不回退到猜测的默认空间。
5. 空间内容服务处于`starting`、`ready`或`unavailable`时分别显示启动中、可进入和暂不可用状态；只有`ready`可以进入工作台可用态。
6. 受控运维入口支持创建本地账号及组织成员关系、创建空间、登记内容服务的`starting`、`ready`或`unavailable`状态、授予或调整空间角色、停用组织或空间、撤销两级成员、停用用户和撤销用户全部会话，使多用户、多空间、不同角色、内容服务状态与撤权不依赖数据库手工预置。

## 2. 同类调研

### 2.1 Docmost

Docmost公开功能包含Spaces、权限和Groups。其自托管初始化会在创建Workspace时同时建立默认Space并授予创建者管理权限；认证后提供空间列表与搜索选择，退出后返回登录入口。

优点是首次使用后立即拥有可进入的知识空间，多空间选择清楚，Workspace与Space权限分层。缺点是自动默认组权限不适合直接照搬到奇点，且Docmost的Workspace语义对应奇点组织，不能继续引入第二套公开命名。

S1借鉴“一次初始化即形成可用空间”和“只从授权列表选择空间”，但初始化只从部署主机的受控运维入口执行，不暴露公开抢占入口，不自动建立全员组权限，也不使用硬编码默认空间作为后续路由事实。

### 2.2 Outline

Outline把登录后的页面集中在认证布局内，并以Team与Collection成员关系决定可见和可写能力。私有Collection只有具备用户或用户组成员关系时可读，未知路由统一进入不存在页面。

优点是认证入口、资源可见性和角色能力边界一致，不可见资源不会通过导航暴露。缺点是Team与Collection不承担奇点的一空间一Kernel隔离语义，Outline的认证方式与奇点S1本地密码也不同。

S1借鉴“认证后再渲染业务壳”“导航只呈现可见资源”和“不可见深链统一按不存在处理”，但仍使用奇点唯一的组织、空间与空间角色模型。

### 2.3 采纳结论

| 决策 | 依据 | 奇点取舍 |
| --- | --- | --- |
| 首次初始化同时创建首空间 | Docmost初始化后立即可用 | 改为部署主机受控运维入口，拒绝公开抢占初始化 |
| 登录后列出授权空间 | Docmost空间列表与搜索 | 仅列活动的两级成员授权，不显示全组织目录 |
| 业务壳置于认证边界内 | Outline认证路由 | 失效后先清理敏感界面，再返回登录 |
| 不可见资源统一隐藏 | Outline私有Collection策略 | 不区分空间不存在、未授权、已停用 |
| 角色决定可见身份 | Docmost与Outline权限模型 | S1显示权威角色，真实写拒绝由具备写链路的阶段证明 |

## 3. 角色与用户故事

### 3.1 角色

- **基础设施管理员**：从部署主机的受控运维入口建立首位组织所有者、账号、空间与成员授权，并可停用账号或撤销其全部会话。
- **组织成员**：使用本地账号登录，只查看所属组织内被授予的活动空间。
- **空间管理员或编辑者**：进入授权空间并看到自己的最新角色，不由浏览器自行提升权限。
- **空间阅读者**：进入授权空间并明确看到`viewer`身份，不被错误表示为可编辑角色。

### 3.2 用户故事

- 作为基础设施管理员，我能从受控运维入口一次完成首位账号、组织和空间初始化，并能继续配置本地账号、空间和成员角色，不需要直接修改数据库。
- 作为基础设施管理员，我能在部署或检查空间内容服务后登记最新服务状态，使成员看到的启动、可用或暂不可用结果来自正式运维路径。
- 作为本地账号成员，我能登录并回到原本要访问的同源空间，不需要重复寻找入口。
- 作为多空间成员，我只看到自己有权访问的空间，并能按名称选择和切换。
- 作为没有空间权限的成员，我看不到目标空间是否存在，也不能通过修改地址进入。
- 作为空间阅读者，我能进入授权空间并看到自己的`viewer`身份，不会被错误表示为`admin`或`editor`。
- 作为会话失效的用户，我会离开敏感工作台并看到重新登录入口，不会继续看到旧空间内容。
- 作为空间成员，我能区分空间正在启动、内容服务暂不可用和浏览器网络失败，并只对可恢复故障执行重试。

## 4. 关键流程

### 4.1 受控初始化与授权配置

空系统由基础设施管理员从部署主机的受控运维入口提交首位本地登录账号的受保护密码输入、组织名称和首空间名称。并发初始化最多一个成功；没有其他成功提交时，校验或事务失败不产生任何数据，系统保持未初始化并允许修正后重试。并发败者在胜者提交后只得到已经初始化的稳定结果，不得继续创建，也不泄露胜者创建的账号、组织或空间信息。

初始化事务一旦成功，后续重试只得到已经初始化的稳定结果，不重复创建账号、组织或空间；即使首次成功结果未送达，重试也不泄露已有账号、组织名称或初始化时间。浏览器、公网API和普通登录页均不提供初始化或自助注册入口。

初始化后，基础设施管理员可从同一受控运维边界创建本地账号及组织成员、创建空间、登记内容服务状态、授予或调整空间角色、停用组织或空间、撤销两级成员、停用用户和撤销用户全部会话。新空间先处于`starting`；部署主机确认内容服务的受保护部署身份与版本后，可将其登记为`ready`或`unavailable`，也可在重新部署时恢复为`starting`。每次操作要么完整生效，要么保持原状态；重复标识、最后一个所有者保护和不存在目标返回稳定结果，密码、会话凭据、受保护部署身份与版本均不在浏览器、普通成员界面或非必要结果中出现。

### 4.2 登录与深链返回

未登录用户访问应用首页时进入登录页；未登录用户访问空间深链时，系统只保留同源目标。账号和密码正确后，单空间用户进入该空间，多空间用户在没有有效深链时进入空间选择页，无空间用户进入无可用空间状态。

账号不存在、密码错误和账号不可用使用相同的登录失败结果。触发频率限制时显示稍后重试，恢复前不能反复提交；页面不说明限制来自账号还是网络地址。

### 4.3 空间选择与进入

空间入口只列出当前用户同时具备活动组织成员关系和活动空间成员关系的活动空间，并显示组织名、空间名和当前空间角色。搜索只在该授权集合内进行，不提供对不可见组织或空间的探测。

进入空间后先取得服务端最新授权与启动状态。只有响应确认相同组织、相同空间且状态为`ready`时进入工作台可用态；客户端地址、上次选择、本地缓存或固定名称都不能替代该确认。

### 4.4 空间切换与故障恢复

用户选择另一个授权空间后，旧空间立即停止接收新操作，新空间在授权与状态确认完成后才成为当前空间。迟到的旧请求不能覆盖新空间名称、角色或状态；新空间失败时不把旧空间伪装成已切换成功。

`starting`状态显示启动中并在页面可见期间有限检查最新状态；`unavailable`和网络失败保留明确故障类别并提供显式重试。重试只重新读取状态，不重复提交用户写入，也不切换到备用空间或内部地址。

### 4.5 退出、到期与撤权

用户主动退出时撤销当前会话，清除认证与空间相关缓存，离开工作台并进入登录页。浏览器后退不能恢复已授权页面或敏感数据。

会话空闲或绝对到期、当前会话被撤销、用户被停用时显示登录已失效；组织或空间停用、组织成员或空间成员被撤销时显示空间不存在；角色变化后必须以最新角色重新进入，降级为`viewer`时立即更新身份展示。认证或权限失效均不能保留可继续操作的旧空间界面。

## 5. 页面与状态

| 状态 | 用户可见结果 | 允许动作 | 禁止结果 |
| --- | --- | --- | --- |
| 登录中 | 保持表单尺寸稳定并显示提交中 | 等待或取消后重试 | 重复提交、泄露账号存在性 |
| 登录失败 | 显示统一的账号或密码错误 | 修改输入后重试 | 区分账号不存在、停用或密码错误 |
| 登录限流 | 显示稍后重试 | 限制解除后提交 | 继续高频提交、暴露限制维度 |
| 空间加载 | 显示与工作台一致的紧凑加载态 | 等待、返回 | 渲染硬编码空间或旧缓存角色 |
| 无可用空间 | 显示尚未获得空间访问权限 | 退出登录 | 猜测默认空间、显示全组织目录 |
| 空间不存在 | 显示找不到该空间 | 返回空间列表 | 区分不存在、未授权或已停用 |
| 空间启动中 | 显示空间正在启动 | 等待、返回、立即重试 | 提前进入工作台可用态 |
| 空间可用 | 显示真实组织、空间和角色对应的工作台 | 进入工作台、切换空间、退出 | 使用路由或缓存伪造当前空间 |
| 内容服务不可用 | 显示内容服务暂不可用 | 显式重试、返回 | 误报登录失效、自动重复写入 |
| 网络失败 | 显示无法连接到服务 | 显式重试、返回 | 默认成功、切换备用地址 |
| 会话失效 | 清除敏感工作台并显示重新登录 | 重新登录 | 继续显示或操作旧空间内容 |

所有页面沿用奇点设计系统和思源视觉语言。登录使用单一表单面板；空间选择使用紧凑列表；工作台状态使用无装饰插画的`Empty`或`Alert`语义，不使用营销式卡片墙。

## 6. 验收标准

| ID | 可判定的用户结果或稳定合同 | 失败风险与关键路径 | 最低充分证据 |
| --- | --- | --- | --- |
| S1-P-01 | 并发首次初始化最多一个成功；没有其他成功提交时，普通失败不产生数据且可修正后重试；并发败者和成功后的重试只返回已经初始化且不泄露现有租户信息，不重复创建数据 | 部署竞态、重复租户或无主数据；首次部署 | 真实数据库运维入口integration |
| S1-P-02 | 受控运维入口可创建本地账号及组织成员、创建空间、登记三种内容服务状态、授予或调整空间角色、停用组织或空间、撤销两级成员、停用用户和撤销用户全部会话；每次操作原子生效且不把密码、会话凭据或受保护部署信息写入非必要结果、回显或日志 | 只能依赖数据库手工预置、部分授权、内容服务状态或撤权无生产路径、凭据或部署信息泄露；访问配置 | 真实数据库运维入口integration + 日志边界 |
| S1-P-03 | 正确账号和密码建立新会话并进入有效同源目标；错误账号、错误密码、停用账号返回相同失败结果且不泄露账号存在性 | 账号枚举或错误跳转；登录 | 真实HTTP contract + 登录页面component |
| S1-P-04 | 登录频率限制同时约束账号与来源；未知账号与真实账号使用相同`429`结果、通用正文和`Retry-After`恢复时间，限制解除后正常登录 | 暴力破解或由限流差异枚举账号；重复登录 | 真实HTTP contract，受控时钟与来源边界 |
| S1-P-05 | 登录成功签发新的当前会话令牌，登录请求携带的任何旧令牌不能固定为新会话；既有其他设备会话不因普通登录或角色变化被隐式撤销 | 会话固定或误伤其他设备；登录 | 真实HTTP Cookie contract |
| S1-P-06 | 会话Cookie固定名为`__Host-singularity_session`，具有`Secure`、`HttpOnly`、`Path=/`、无`Domain`和`SameSite=Lax`属性，浏览器脚本不能读取 | 凭据窃取、降级传输或跨站滥用；全部认证请求 | 真实HTTP Cookie contract |
| S1-P-07 | 登录用户只取得自己同时具备活动组织成员与空间成员关系的活动空间；无空间时看到明确空状态 | 空间目录泄露或错误默认空间；首页 | 真实HTTP授权合同 + 空间入口component |
| S1-P-08 | 不存在、未授权、组织停用、空间停用和成员撤销的深链都显示同一空间不存在结果，响应不泄露资源差异 | 资源枚举或越权进入；深链 | 真实HTTP contract + 路由component |
| S1-P-09 | 空间入口只在服务端确认相同组织与空间后采用返回角色和状态；地址、缓存或固定值不能创建可用工作台 | 跨空间会话或客户端自证权限；空间进入 | HTTP contract + React路由component |
| S1-P-10 | 授权启动结果只公开组织、空间、角色和内容服务状态，不包含内部实例、地址、路径、凭据或同义空间字段 | 内网与部署信息泄露；空间进入 | 公共响应contract |
| S1-P-11 | `admin`、`editor`和`viewer`返回并显示各自最新权威角色；角色变化后旧角色不再显示，S1不以尚不存在的内容写入口证明真实写拒绝 | 角色漂移或浏览器自证权限；角色消费 | 真实HTTP授权合同 + 路由component |
| S1-P-12 | 受控运维入口登记的`starting`、`ready`和`unavailable`产生三种明确状态，只有`ready`进入工作台可用态；网络失败不冒充内容服务故障，测试不得直接写数据库制造状态 | 错误恢复、空白页、错误身份提示或测试绕过生产路径；空间启动 | 真实运维入口 + HTTP contract + 路由component |
| S1-P-13 | 切换空间时旧结果不能覆盖新空间，失败切换不会显示为成功，且任一时刻只有一个当前空间 | 跨空间污染或竞态；空间切换 | React路由component状态转移 |
| S1-P-14 | 主动退出只撤销当前会话并清除认证与空间缓存；真实浏览器后退、历史恢复和页面缓存不能恢复已授权界面 | 退出后内容残留；退出 | 真实HTTP撤销与缓存contract + browser integration |
| S1-P-15 | 30分钟空闲或12小时绝对到期、当前会话撤销、运维撤销全部会话和用户停用均使后续访问进入重新登录状态；旧会话不能恢复 | 过期或已撤销凭据继续访问；会话生命周期 | 真实HTTP contract，受控时钟与运维入口integration |
| S1-P-16 | 组织或空间停用、两级成员撤销使受影响访问变为统一不存在；角色变化后的下一授权结果只返回新角色 | 撤权延迟或陈旧权限；权限变化 | 真实数据库HTTP integration + 路由component |
| S1-P-17 | 登录拒绝缺失或错误的精确Origin；已登录的全部非安全HTTP方法同时拒绝缺失或错误的精确Origin及CSRF凭据，拒绝后状态不变；受控运维入口不经HTTP暴露 | 跨站登录或请求伪造；登录与退出 | 真实HTTP security contract |
| S1-P-18 | 登录、空间选择、空状态和故障状态在桌面与移动视口无文字溢出或控件重叠，表单可用键盘完成并具有可访问名称与焦点顺序 | 无法登录或选择空间；核心入口 | component可访问性 + browser integration |

S1交付必须由标准强制入口发现真实数据库运维操作、真实HTTP合同、React组件、真实浏览器行为与production build证据。永久测试不得用静态源码字符串、私有方法、完整内部mock或静态壳截图冒充运行时结果；具体runner、命令和后续阶段分工由架构与测试方案定义。

## 7. 约束

- **安全**：本地密码采用抗暴力破解摘要；登录失败不区分账号、密码和停用状态；敏感凭据不进入URL、浏览器持久存储、日志或错误正文。
- **运维边界**：初始化与最小访问配置只在部署主机的受控运维入口执行，不经浏览器或公网HTTP暴露；普通成员界面不能调用该能力。
- **权限**：组织成员关系和空间成员关系必须同时成立；浏览器显示的角色不是服务端授权替代品。
- **隐私**：不可见资源、停用资源和不存在资源使用同一外部结果；空间列表不返回未授权名称、数量或搜索命中。
- **会话**：当前会话具有空闲与绝对期限；退出、到期和撤销后敏感查询与空间状态不得从缓存恢复。
- **兼容**：保持服务器权威与一空间一Kernel方向，不增加本地空间、离线模式、旧思源标识或`workspaceId`同义字段。
- **视觉**：遵循奇点设计系统的6px圆角、紧凑密度、语义色、shadcn组件和思源Daylight风格；不建立平行视觉系统。
- **可访问性**：核心入口达到WCAG AA文本对比度，输入具备Label，图标按钮具备可访问名称，键盘焦点可见且顺序与视觉一致。
- **响应式**：支持最小320px宽度；固定控件不因加载、长组织名、长空间名或错误文字改变整体布局。
- **文案**：使用用户可理解的登录、空间与服务状态，不显示内部模块名、路由策略、运行时对象、测试阶段或未来路线。
- **测试价值**：永久测试保护登录、会话、权限、资源隐藏和状态转移等稳定合同；失败时应主要指向一个用户结果，且由标准聚合入口发现。
