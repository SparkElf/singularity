# L3.1 生产实时协作实施计划

## Objective

按以下权威文件实现并验证奇点 L3.1 生产实时协作：

- 产品：[docs/product/l3.1-realtime-collaboration.md](../docs/product/l3.1-realtime-collaboration.md)
- 架构：[docs/architecture/l3.1-realtime-collaboration.md](../docs/architecture/l3.1-realtime-collaboration.md)
- ADR：[docs/adr/0031-l3-production-collaboration.md](../docs/adr/0031-l3-production-collaboration.md)
- L3.0 门禁：[docs/verification/l3-realtime-collaboration.md](../docs/verification/l3-realtime-collaboration.md)

首期部署固定为单 API 副本、单空间 Kernel 协作通道；加密内容库只有显式受限协作、文档授权和 Kernel 密钥可用时才允许加入。

## Current State

- L0、L1、L2 已完成并同步 `origin/master`。
- L3.0 语义原型已通过既有验证；L3.1 生产 contracts、React/Protyle host、专用 WSS、Nest coordinator、Go Kernel bridge/history 和 Prisma 控制面元数据已接入，默认开关仍关闭，已通过 code-review、test-governance 复评和集中 verification。
- 现有 Protyle WebSocket 只接收服务端推送；正式内容写入走受策略保护的 Kernel HTTP 事务。
- 现有 L2 `DocumentAccessPolicy`、`AccessChanged`、`SpaceConnectionRegistry`、审计和 React/shadcn/Tailwind 4 设计系统是本阶段先例。

## Implementation Progress

- Kernel undo/redo 通过既有 `isUndoReplay` 事务上下文回到 Protyle push 消费点；普通正文 transactions 在协作会话期间继续过滤，避免双路径。
- Kernel production coordinator 已使用 `.wal` prepared/committed 日志包住真实内容事务；启动时只自动恢复 committed 日志，prepared 或未知阶段 fail-closed 为 `ErrCollaborationRecoveryRequired`。
- Protyle mapper 已覆盖纯文本最小差分、块插入索引和富文本显式拒绝；广播必须通过四段身份合同。
- 协作客户端已隔离旧 socket 迟到事件，operation-result 才确认提交，手动关闭会拒绝并清理 pending 操作。
- Nest 协调器已绑定认证用户、撤权后立即移除活跃会话，空间 Gateway freeze 会解除协作广播订阅；这些状态边界已补永久单元测试。
- 当前 implementation 代码、调用方、迁移、永久测试和静态门禁已完成；code-review、test-governance 复评和唯一 aggregate 均已通过，进入灰度准备。

## Locked Contracts

- Go Kernel 是正文、块、AV、历史和 canonical operation log 的唯一事实源。
- PostgreSQL 不保存第二份正文、CRDT 快照或 operation payload；只保存协作开关、会话元数据、ACL 和审计投影。
- 所有 join、submit、resume、presence、确认、冲突、撤权和关闭消息显式携带 `organizationId + spaceId + notebookId + documentId`。
- 专用双向协作 WSS 与只接收推送的 Protyle WSS 分离；不增加浏览器到 Protyle 的命令帧。
- `DocumentAccessPolicy` 是 ACL 唯一 owner；协作协调器不重做用户组/角色算法，只消费 capability 和撤权代次。
- 首期单副本不引入 Redis、NATS、BullMQ 或跨副本兼容分支；多副本另立 ADR。
- 加密库默认关闭实时协作；受限协作必须经过成员、文档 ACL、内容密钥和客户端版本四项 admission。密钥不进入控制面、日志或浏览器持久化。
- 禁止整文档快照覆盖、锁、LWW、静默丢弃、明文 fallback 和隐式身份推断。
- 后端使用 Nest 声明式 `@Module`、DI provider、Guard、Pipe、Interceptor 和 operation handler metadata；同一 operation kind/version 冲突导致启动失败。
- 关键跨模块函数必须添加必要中文备注，说明前置条件、后置状态和副作用；异常日志保留原始 `name/message/stack`，不记录正文/密钥/token。

## Stage Boundary

L3.1 是一个完整的大阶段，包含生产 contracts、Kernel bridge、Nest 协作 Gateway、单副本生命周期、加密 admission、React/Protyle host、功能开关、永久测试、旧路径清理和文档。全部模块完成、旧路径清理、代码评审和 test-governance 复评通过后，已统一执行一次 verification aggregate；本计划现进入已验证状态。

## Module Owners and File Scope

当前不使用子代理。由于公共 contracts、单副本连接注册表和 Kernel history 有强顺序依赖，模块按纵向合同串行集成；每个模块完成自身生产代码、类型、调用方、迁移、测试和文档。

| 顺序 | 模块 owner | 文件范围 | 前置与交接合同 |
| --- | --- | --- | --- |
| 1 | L3.1 contracts | `enterprise/packages/contracts/src/realtime-collaboration.ts`、聚合导出、paths/OpenAPI | 冻结四段身份、操作、冲突、加密拒绝码和 session generation |
| 2 | Kernel production bridge | `kernel/collab/**`、history/transaction bridge、Go fixtures | 消费解析后的语义操作；输出 canonical result/history；不做 HTTP/ACL |
| 3 | Nest collaboration | `enterprise/apps/api/src/collaboration/**`、CoreModule/metadata、WSS adapter | 消费 contracts 和 Kernel port；实现 join/submit/resume/presence/revoke |
| 4 | Control-plane persistence | `enterprise/packages/database/**`、feature flag/session metadata/audit | 只持久化控制面元数据；复合身份和撤权事实来自既有 ACL |
| 5 | React/Protyle host | `enterprise/apps/web/src/**`、设计系统 variants、Zustand store | 消费专用 WSS；只保存会话、冲突和 presence，不复制正文 |
| 6 | Integration and closure | 根 `package.json`、唯一 aggregate、static audits、docs | 在所有模块完成后统一接线、删除旧入口并收口证据 |

共享文件由 integration owner 统一修改；如出现并行修改冲突，先在 `/root/projects/mailbox.md` 记录未决文件和 owner，处理后立即删除对应消息，不覆盖其他线程改动。

## Declarative Backend Contract

| 声明 | 参数合同 | 发现范围 | 冲突/失败 | 可观测证据 |
| --- | --- | --- | --- | --- |
| `@Module` + DI providers | `KernelGatewayModule` 中的协作 port、coordinator、Gateway 和 feature gate | `KernelGatewayModule.register()` provider scope | 缺少 provider 在 Nest 装配阶段失败 |真实 Nest bootstrap |
| `@HandlesCollaborationOperation(kind, version)` | 语义 kind/version 和 handler method metadata | `CollaborationOperationDiscovery` + `DiscoveryService` | 同 kind/version 多 provider 启动失败 |真实 metadata discovery + handler result |
| `@Authenticated`/`@SessionMutation` + Zod Pipe | HTTP 会话、写入 mutation 和公开 schema | feature controller route metadata | 未认证或 schema 不符由既有 Guard/Pipe 拒绝 |真实 HTTP contract |

声明只负责装配、策略和横切元数据；排序、事务、冲突和加密状态转移仍由显式 service/use case 负责。不得引入中央 switch、手工 registry、源码扫描或 decorator factory 隐藏 I/O。

## Boundary Ownership

| 边界 | 首次 owner | 输入/输出 | 下游不重复处理 |
| --- | --- | --- | --- |
| 浏览器到 WSS | protocol parser | JSON/binary -> typed envelope | service 消费已解析四段身份和 operation |
| 当前会话到 ACL | CollaborationAccessGuard + L2 policy | session -> capability/generation | coordinator 不重做 ACL 角色计算 |
| 协作到 Kernel | KernelCollaborationPort | typed operation -> canonical result | Kernel reducer 不解析 HTTP/WS 或重新授权 |
| Kernel 到 history | Go Kernel history owner | accepted operation -> append/replay | PostgreSQL 不保存正文或操作 payload |
| 加密 admission | Kernel content session | key available -> apply / stable rejection | Gateway 不持有或推断密钥 |
| React 到 Protyle | ProtyleHost semantic port | typed result -> DOM/editor transaction | Zustand 不保存正文快照 |

## Verification Matrix

唯一正式命令：`cd /root/projects/singularity/enterprise && pnpm verify:l3-production`。顺序固定为 contracts -> Go Kernel -> API/WS/PostgreSQL -> security/encryption -> React/browser -> performance -> static/build。标准 runner 必须支持独立 case、hook cleanup、按名称过滤和结构化报告。

| 合同 | 层级 | 证据边界 | 主要失败归因 |
| --- | --- | --- | --- |
| schema、四段身份、拒绝码、handler metadata | contract/static | 真实序列化、Nest bootstrap、结构化依赖审计 | 合同漂移或声明装配错误 |
| reducer、inverse、冲突、history replay | Go unit/integration | 真实 Kernel 内容夹具 | 语义丢失或重放漂移 |
| join/submit/resume/ACL revoke/单副本 restart | API/WS integration | 真实 Nest、固定 PostgreSQL、受控 Kernel | 旧 grant、迟到消息、假恢复 |
| encrypted admission、密钥/正文不落控制面 | security/integration | 真实内容授权边界、logger/storage audit | 明文/密钥泄露 |
| React session、冲突、只读、320px | component/browser integration | 真实组件和浏览器；外部 Kernel 仅在明确边界替换 | 状态覆盖、溢出、身份错绑 |
| 上限、关闭开关、资源释放 | performance/integration | 固定客户端/操作数据集 | 无限队列、残留连接 |
| 无生产原型路径、无旧 WS 写入、无正文表 | static/AST/config | 依赖方向、配置和禁用 API | 边界泄漏 |
